Возможности

Полный набор функций удалённого доступа для личного использования.

Захват экрана

MediaProjection + аппаратный кодировщик H.264. До Full HD, 30 FPS, адаптивное качество в зависимости от скорости сети.

Удалённое управление

Тап, двойной тап, удержание, свайпы, прокрутка, ввод текста, кнопки «Назад», «Домой» и список последних приложений.

Передача файлов

В обе стороны, файлы более 2 ГБ, возобновление передачи после обрыва соединения.

Безопасность

TLS/DTLS, AES‑256, авторизация по коду подключения, подтверждение первого входа и белый список доверенных устройств.

Сеть WebRTC

STUN/TURN, автоматический выбор оптимального маршрута и восстановление соединения без разрыва сессии.

Мультисессии

Десктоп-клиент для Windows: тёмная и светлая темы, избранные устройства и до 20 одновременных подключений.

Системные требования

Android-агент

  • Android 8.0 (API 26) и выше
  • Поддержка MediaProjection и Accessibility Service
  • Совместимость: Samsung, Xiaomi, Redmi, Poco, Oppo, Vivo, OnePlus, Realme, Google Pixel
  • Разрешения на захват экрана и спец. возможности

Windows-клиент

  • Windows 10/11 (64‑бит)
  • 4 ГБ ОЗУ и выше
  • Подключение к интернету
  • Поддержка нескольких мониторов и полноэкранного режима

Сервер

  • VPS на Ubuntu 22.04 или Debian 12
  • Docker и docker‑compose
  • coturn (STUN/TURN)
  • Открытые порты сигналинга и TURN

Свой сервер

Orbit Remote работает на вашей инфраструктуре. Сигнальный сервер и TURN разворачиваются одной командой через docker‑compose. Никакие медиаданные не проходят через сторонние сервисы — соединение устанавливается напрямую между устройствами.

  • Регистрация устройств и хранение ID
  • WebRTC‑сигналинг по WebSocket
  • Готовые конфигурации STUN/TURN (coturn)
  • Логирование подключений: время, IP, длительность сессии
# Развёртывание на VPS
git clone <your-repo>
cd orbit-remote/server
cp .env.example .env   # укажите домен и секреты
docker compose up -d

# Проверка
curl https://your-domain/health
